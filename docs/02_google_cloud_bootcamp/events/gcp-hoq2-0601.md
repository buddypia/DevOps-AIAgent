## Description

「エージェント」という言葉が一人歩きしがちな今、まずは AI エージェントを構成する 3 つの要素（モデル・ツール・システム プロンプト）に立ち返り、どんな課題に対してエージェント化が有効なのかを整理します。その後、Google Cloud が提唱する代表的な設計パターンの分類（単一とマルチ、決定論的と動的、反復処理）を俯瞰し、最もシンプルなシングル エージェントをハンズオンで実装します。「とりあえず動く」から「設計判断ができる」への第一歩となる回です。
